Carrier

One explicit envelope.

The official README describes a UTF-8 JSON Mezcalstone pushed immediately after OP_RETURN. The current decoder finds the nulldata output, extracts its payload, parses JSON, then strict-validates the result.

Official carrier shape
OP_RETURN <utf8-encoded-mezcalstone-json>
From the official README. There is no OP_13 marker.
Accepted protocol identifiers
"mezcal"
"https://mezcal.sh"
"https://t.me/mezcalbtc"
These are the three literal p values accepted by the current MezcalstoneSchema.
Official code excerpt

What the validator accepts.

This compact TypeScript excerpt is taken from the official validator. It is the decisive clue that an extra, unknown JSON key is not harmless: the schema ends in .strict().

src/lib/mezcalstone.ts, current pinned source
export const MezcalstoneSchema = z.object({
  p: z.union([
    z.literal("mezcal"),
    z.literal("https://mezcal.sh"),
    z.literal("https://t.me/mezcalbtc"),
  ]),
  edicts: z.array(EdictSchema).optional(),
  etching: EtchingSchema.optional(),
  mint: MintSchema.optional(),
  pointer: u32().optional(),
}).strict();
Source: bitapeslabs/mezcal @ 0f3323f.
Payload gallery

Start from valid JSON.

These payloads are shaped around the current schema. Replace IDs, addresses, and final output indexes with values from your own finalized transaction.

Etch an asset
{
  "p": "https://mezcal.sh",
  "etching": {
    "divisibility": 8,
    "premine": "0",
    "mezcal": "taco",
    "symbol": "🌮",
    "terms": null,
    "turbo": true
  }
}
The schema permits names of 1 to 15 lowercase letters, digits, and hyphens. Divisibility is 0 through 18.
Mint and allocate
{
  "p": "mezcal",
  "mint": "840000:1",
  "edicts": [
    { "id": "840000:1", "amount": "250", "output": 1 }
  ],
  "pointer": 2
}
An edict can also use the tuple form ["840000:1", "250", 1]. Output indexes must exist in the final transaction.
Priced flex mint
{
  "p": "https://mezcal.sh",
  "etching": {
    "divisibility": 8, "premine": "0",
    "mezcal": "wbtc", "symbol": "₿",
    "terms": {
      "price": [{ "amount": "1", "pay_to": "bcrt1p..." }],
      "amount": "0", "cap": null,
      "height": [0, null], "offset": [null, null]
    }, "turbo": true
  }
}
Based on the official flex-mint example. pay_to must be a valid address for the active network.
Known failure path
invalid JSON
  | invalid schema
  | bad edict output index
  | block 0 with nonzero tx index
= cenotaph
The reference decoder treats invalid Mezcalstones as a cenotaph. Preflight the exact serialized transaction, not a draft object.
Field ledger

Small fields, hard limits.

Amount values are decimal strings for u128 range. Output indexes, asset IDs, and date windows have distinct integer limits that should be validated before wallet approval.

FieldCurrent validator ruleWhy it matters
pOne of three literals: mezcal, https://mezcal.sh, or https://t.me/mezcalbtc.Unknown protocol headers fail strict validation.
mezcal1-15 lowercase a-z, digits, or hyphens.Name is part of the etching identity and must be unique in protocol state.
divisibilityInteger 0 through 18.Controls display precision, not the internal u128 amount representation.
premine / amount / capNon-empty decimal strings inside unsigned 128-bit range.Do not use JSON numbers or floating point conversion.
mint / edict idblock:tx, both valid unsigned 32-bit integers.Identifies the existing asset or allocation target.
edict outputUnsigned 8-bit integer, no larger than final output count minus one.Determines where the asset is allocated.
pointerUnsigned 32-bit output index.Receives remaining unallocated amount.
terms.priceOne or more amount and pay_to pairs.Every priced mint must satisfy the required payment outputs.
Builder warning. Freeze all transaction outputs before deriving edict or pointer values. An output added for fees, change, or payment after indexes are chosen changes the meaning of the Mezcalstone.
