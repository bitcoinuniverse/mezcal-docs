CLI fast path

Use the official commands.

These commands come from the official Mezcal CLI README. They give a wallet-connected starting point for discovery, etching, minting, and transfers.

Install and orient
npm install -g mezcal-cli
mezcal --help

mezcal info 859:1
mezcal balance
mezcal etch
mezcal mint 859:1
mezcal transfer
Replace 859:1 with a real asset ID or use a Mezcal name when supported.
Production path

Six checks before a real transaction.

Tooling can construct a transaction, but it cannot rescue a payload that points to the wrong output. Keep each step observable and repeatable.

Pin the protocol version.

Record the Mezcal reference commit and CLI version you validated against. The source links in this site are pinned for this reason.

Model outputs before JSON.

Decide payment, recipient, asset change, and pointer destinations before you write an edict output index.

Represent amounts as strings.

Use non-empty decimal strings for every u128 amount. Never round through JavaScript number or floating point arithmetic.

Validate strict schema locally.

Confirm the accepted p literal, field types, address rules, name grammar, limits, and absence of unknown keys.

Decode the final transaction.

Re-extract the OP_RETURN payload from the signed transaction and compare byte-for-byte with the intended UTF-8 JSON.

Verify indexed allocations.

After confirmation, query a Mezcal-aware indexer and reconcile asset amounts by output, including any pointer remainder.

Release gate

Do not broadcast until all pass.

These checks target the exact failure paths in the reference decoder and the operational hazards of output-indexed asset allocation.

  • Every JSON key is accepted by the strict Mezcalstone schema. No debug fields, UI metadata, or unknown extension fields remain.
  • All amounts are decimal strings in u128 range, and divisibility is 0 through 18.
  • Each block:tx identifier and every edict output index passes its integer range and transaction-shape check.
  • Every terms.price requirement maps to a final payment output with the required amount and recipient.
  • Final transaction output order is frozen before calculating edicts and pointer.
  • The signed transaction decodes to exactly the intended JSON and the indexer reports the expected allocation after confirmation.
Cenotaph risk. The reference decoder marks malformed JSON, schema failures, invalid edict output indexes, and invalid zero-block IDs as cenotaph cases. Treat that outcome as a hard stop in your preflight.
